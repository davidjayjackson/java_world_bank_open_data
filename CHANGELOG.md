# Changelog

All notable changes to the World Bank Open Data LibreOffice Calc add-in are
documented here.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - 2026-07-16

First release. A Java UNO add-in (`com.sun.star.sheet.AddIn`) exposing the
[World Bank Open Data Indicators API](https://api.worldbank.org/v2/) as
LibreOffice Calc worksheet functions, packaged as `WorldBank.oxt`. No API
key required.

### Added
- Worksheet functions: `WBVALUE`, `WBSERIES`, `WBLATEST`, `WBMETA`,
  `WBLASTERROR`, `WBCACHECLEAR`.
- Non-blocking cell functions: a shared, TTL'd, background-refreshed cache
  (`WorldBankCache`) so no formula ever blocks on network I/O. Functions
  return `#FETCHING`, `#NOT_FOUND`, or `#ERR` sentinels instead of throwing;
  `WBLASTERROR()` surfaces the last failure's detail.
- `WBVALUE`, `WBSERIES`, and `WBLATEST` for the same (country, indicator)
  pair share a single full-history fetch and cache entry, rather than each
  issuing its own `date=`/`mrv=` request.
- Correct handling of the World Bank API's two-element-array response shape
  (pagination/error header at index 0, observations at index 1, which may
  be `null`), the `message`-array error shape for invalid indicator/country
  codes, and null observation values (mapped to blank cells, never `0`).
- `CompatibilityName` set for every function in `CalcAddIns.xcu`, so
  formulas survive an XLS/XLSX save-as/reopen round trip.
- Build pipeline (`build.sh`: `unoidl-write` → `javamaker` →
  `javac --release 8` → `jar` → `.oxt`) and a headless live smoke test.
- Demo workbook (`test/worldbank_demo.ods`, regenerated by
  `tools/build_demo.py`) built around US/UK GDP.
- MIT license.

### Notes
- Pure JDK implementation: `HttpURLConnection` + a hand-rolled JSON parser,
  no third-party jars. Compiled to Java 8 bytecode so it runs on the JRE 8
  LibreOffice accepts by default.
- Verified live against the real World Bank API: `WBVALUE` (explicit year
  and "most recent"), `WBLATEST`, `WBMETA`, `WBSERIES` (bounded range, full
  history, and multi-country) and the `#ERR`/`WBLASTERROR` path for an
  invalid indicator code all confirmed working end-to-end against a
  headless LibreOffice instance.
- `WBCACHECLEAR`'s returned count is best-effort: LibreOffice's formula
  engine does not guarantee a cell's function is invoked exactly once per
  recalculation, so a second, silent invocation of that cell can see an
  already-drained cache and report 0 even though entries were cleared.
